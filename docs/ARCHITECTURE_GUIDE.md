# ARCHITECTURE_GUIDE.md — 쉽게 읽는 구조 안내 (입문자용)

> 대상 독자: 프로그래밍을 잘 모르는 분(일반 직장인·학부생 수준).
> 개발자용 상세 문서는 `ARCHITECTURE.md`를 보세요.

## 이 앱은 무엇인가요?

L-Acoustics, d&b audiotechnik, Meyer Sound 세 회사의 스피커·앰프·
프로세서(DSP)·소프트웨어·액세서리 사양을 한 화면에서 검색하고 비교하는
웹사이트입니다. 설치 프로그램 없이 브라우저에서 열리는 단일 페이지
앱이며, 별도의 서버나 데이터베이스 없이 모든 정보가 프로젝트 안의
데이터 파일에 들어 있습니다.

## 화면 구성

- 상단 탭: Speaker / Amplifier / DSP / Software / Accessories / Brand.
  탭마다 제품 카드가 격자로 나열됩니다.
- 검색창·필터: 이름으로 검색하거나(띄어쓰기·하이픈이 달라도 찾아줍니다.
  예: "KS 28", "ks-28", "KS28" 모두 동일), 제조사·타입 등의 칩 버튼과
  범위 슬라이더로 좁혀 볼 수 있습니다.
- 카드 클릭 → 상세 팝업(모달): 사진, 사양 표, 어울리는 앰프 목록 등.
- 나란히 보기(Split View): 상세 팝업 안에서 연관 제품(예: 매칭 앰프)을
  클릭하면 화면이 좌우로 나뉘어 두 제품을 동시에 비교할 수 있습니다.
  사진을 클릭하면 오른쪽에 크게 확대해 볼 수도 있습니다.
- 주소(URL) 공유: 상세 팝업이나 나란히 보기 상태 그대로 주소가 바뀌므로,
  그 주소를 복사해 보내면 받은 사람도 같은 화면을 바로 열 수 있습니다.
  브라우저 뒤로가기를 누르면 팝업이 닫힙니다.

## 폴더는 이렇게 나뉘어 있어요

| 위치 | 내용 |
|---|---|
| `index.html` | 웹사이트의 시작 파일 |
| `README.md` | 프로젝트 전체 안내서 (가장 먼저 읽는 문서) |
| `docs/` | 문서 모음 (이 파일, 기술 문서, 업데이트 기록) |
| `raw-data/` | 제조사 원문 스펙 문서와 원본 사진 보관함 |
| `public/assets/` | 웹사이트에 실제로 쓰는 사진과 글꼴 |
| `public/css/` | 화면 디자인(색·배치) 파일들 |
| `public/js/` | 동작을 담당하는 코드 |
| `public/tests/` | 코드가 잘 동작하는지 자동으로 검사하는 파일들 |

## 코드는 이렇게 나뉘어 있어요 (`public/js/`)

- `domains/` — 제품 종류(도메인)별 폴더. 예를 들어 `domains/speakers/`
  안에는 네 가지 파일이 있습니다:
  - `…data.js` 제품 정보(이름, 무게, 사진 경로 등)
  - `…schema.js` 검색·필터 규칙("어떤 항목으로 거를 수 있나")
  - `…view.js` 카드와 팝업의 생김새(HTML 조각 생성)
  - `…controller.js` 위 세 가지를 연결해 실제로 동작시키는 지휘자
- `core/` — 모든 도메인이 함께 쓰는 기본 부품(주소 처리, 검색 엔진 등)
- `ui/` — 모든 도메인이 함께 쓰는 화면 부품(팝업, 카드 격자, 필터 패널 등)

**새 제품을 추가하려면** 해당 도메인의 `data/` 폴더 파일에 항목을 더하면
되고, 화면 코드는 건드릴 필요가 없습니다. 원문 스펙 문서는 반드시
`raw-data/raw-specs/`에 함께 보관합니다(규칙: 루트 `README.md` 참고).

## 자주 쓰는 명령

- 실행: 프로젝트 폴더에서 정적 서버 실행 후 브라우저로 접속
  (예: `npx serve .`)
- 검사: `npm test` — 데이터 무결성과 화면 동작을 자동 점검합니다.
